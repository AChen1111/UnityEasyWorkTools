# UnityEasyWorkTools TableImporter 说明书

## 简介

TableImporter 是 UnityEasyWorkTools 的 Unity Editor 导表模块, 用于把 `.xlsx` 或 `.csv` 表格导入为 ScriptableObject 资产。它适合把策划表、配置表、掉落表、物品表、武器表等编辑期数据转换为 Unity 运行时可以直接读取的资产。

通用导表核心位于 `Editor/Core`, 和具体游戏类型绑定的 importer 脚本位于 `Importers` 并通过单独 asmdef 引用项目程序集。

## 安装

把 `Assets/UnityEasyWorkTools` 目录放入项目即可使用。当前项目已经把 TableImporter 放在 `Assets/UnityEasyWorkTools/TableImporter`。

路径配置统一放在:

```text
Assets/UnityEasyWorkTools/UnityEasyWorkToolsPathSettings.asset
```

可以通过 `Tools/UnityEasyWorkTools/Settings/Open Path Settings` 打开并修改默认路径。

## 推荐项目结构

```text
Assets/
  UnityEasyWorkTools/
    TableImporter/
      Editor/
        Core/
        UI/
      Importers/
        ItemDatabaseExcelImporter.cs
        WeaponDatabaseExcelImporter.cs
      Docs/
        说明书.md
```

`Editor` 目录保留通用逻辑。`Importers` 目录负责引用项目自己的 ScriptableObject 类型。

## 创建导入器

导入器一般放在 `Assets/UnityEasyWorkTools/TableImporter/Importers`。如果项目启用了 asmdef，需要让 importer 程序集引用 `Excel2SO.Editor` 和对应项目运行时程序集。

示例:

```csharp
using UnityEngine;

public sealed class ItemDatabaseExcelImporter : Excel2SoListAssetImporter<ItemDatabase>
{
    protected override string DefaultAssetPath => "Assets/Resources/ItemDatabase.asset";

    protected override string ListPropertyPath => "items";

    protected override void Configure(Excel2SoMapping map)
    {
        map.Column("itemId").To("itemId").AsInt();
        map.Column("itemName").To("itemName").AsString();
        map.Column("description").To("description").AsString();
        map.Column("icon").To("icon").AsAsset<Sprite>();
    }
}
```

## 表格格式

第一行作为表头。后续每一行作为数据行。

示例:

```csv
itemId,itemName,description,icon
1,Health Potion,Restore health,Assets/Icons/health.png
2,Coin,Money item,Assets/Icons/coin.png
```

表头名称需要和 importer 中 `map.Column("列名")` 的列名一致。空行会被跳过。

## 支持的映射类型

常用映射:

- `AsString()`
- `AsInt()`
- `AsFloat()`
- `AsBool()`
- `AsVector2()`
- `AsVector3()`
- `AsColor()`
- `AsEnum<TEnum>()`
- `AsAsset<TAsset>()`
- `AsStringList()`
- `AsIntList()`
- `AsFloatList()`
- `AsAssetList<TAsset>()`

列表默认可以用分隔符拆分。项目中常用 `;` 作为列表分隔符。

## 使用编辑器窗口

在 Unity 菜单打开:

```text
Tools/UnityEasyWorkTools/Table Importer/Importer Window
```

基本流程:

1. 选择 importer 类型。
2. 选择 `.xlsx` 或 `.csv` 表格。
3. 选择或确认目标 ScriptableObject 资产。
4. 点击导入。
5. 查看 Console 中的导入报告。

## 代码生成

编辑器窗口提供代码生成入口，可以生成基础数据类、数据库类和 importer 模板。生成后建议把运行时数据脚本留在项目自己的 `Assets/Scripts`, importer 模板留在 `Assets/UnityEasyWorkTools/TableImporter/Importers` 中。

默认 importer 生成目录建议使用:

```text
Assets/UnityEasyWorkTools/TableImporter/Importers/Generated
```

## 和 asmdef 一起使用

如果项目专用 importer 独立成 asmdef, 需要添加引用:

```json
"Excel2SO.Editor"
```

这样项目 importer 才能访问 `Excel2SoListAssetImporter<TAsset>`、`Excel2SoMapping`、`ExcelTable` 等类型。当前项目使用 `UnityEasyWorkTools.TableImporter.Importers.Editor.asmdef` 管理这些引用。


## 常见问题

### 找不到 Excel2SoListAssetImporter

检查项目 Editor asmdef 是否引用了 `Excel2SO.Editor`。

### 编辑器窗口找不到 UXML 或 USS

检查 `Assets/UnityEasyWorkTools/TableImporter/Editor/UI` 下是否存在 `Excel2SoImporterWindow.uxml` 和 `Excel2SoImporterWindow.uss`。

### importer 找不到项目里的数据类型

检查 `UnityEasyWorkTools.TableImporter.Importers.Editor.asmdef` 是否引用了目标数据类型所在的运行时程序集。
