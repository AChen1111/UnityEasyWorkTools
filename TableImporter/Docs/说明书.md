# TableImporter 说明书

TableImporter 是 UnityEasyWorkTools 的 Unity Editor 导表模块, 用于把 `.xlsx` 或 `.csv` 表格导入为 ScriptableObject 资产。

## 安装与配置

把仓库放入 Unity 项目的以下目录:

```text
Assets/UnityEasyWorkTools
```

路径配置资产:

```text
Assets/UnityEasyWorkTools/UnityEasyWorkToolsPathSettings.asset
```

菜单入口:

```text
Tools/UnityEasyWorkTools/Settings/Open Path Settings
```

## 创建导入器

导入器一般放在 `Assets/UnityEasyWorkTools/TableImporter/Importers`。如果项目启用了 asmdef, importer 程序集需要引用 `Excel2SO.Editor` 和目标 ScriptableObject 所在程序集。

示例:

```csharp
using UnityEngine;

public sealed class ItemDatabaseExcelImporter : Excel2SoListAssetImporter<ItemDatabase>
{
    protected override string DefaultAssetPath => "Assets/Resources/ItemDatabase.asset";

    protected override string ListPropertyPath => "items";

    protected override void Configure(Excel2SoMapping map)
    {
        map.Column("itemId").To("itemId").AsInt();
        map.Column("itemName").To("itemName").AsString();
        map.Column("description").To("description").AsString();
        map.Column("icon").To("icon").AsAsset<Sprite>();
    }
}
```

## 表格格式

第一行作为表头, 后续每一行作为数据行。表头名称需要和 `map.Column("列名")` 保持一致。

```csv
itemId,itemName,description,icon
1,Health Potion,Restore health,Assets/Icons/health.png
2,Coin,Money item,Assets/Icons/coin.png
```

## 支持的映射类型

- `AsString()`
- `AsInt()`
- `AsFloat()`
- `AsBool()`
- `AsVector2()`
- `AsVector3()`
- `AsColor()`
- `AsEnum<TEnum>()`
- `AsAsset<TAsset>()`
- `AsStringList()`
- `AsIntList()`
- `AsFloatList()`
- `AsAssetList<TAsset>()`

列表默认支持 `;`、`,`、`|` 作为分隔符。

## 使用编辑器窗口

菜单入口:

```text
Tools/UnityEasyWorkTools/Table Importer/Importer Window
```

基本流程:

1. 选择 importer 类型。
2. 选择 `.xlsx` 或 `.csv` 表格。
3. 选择或确认目标 ScriptableObject 资产。
4. 点击导入。
5. 查看 Console 中的导入报告。

## 代码生成

编辑器窗口可以生成数据类、数据库类、importer 模板和 CSV 模板。生成目录由 `UnityEasyWorkToolsPathSettings.asset` 管理。

建议目录:

```text
Assets/Scripts/Generated/Excel2SO
Assets/UnityEasyWorkTools/TableImporter/Importers/Generated
Assets/csv
```

## asmdef 配置

项目 importer 独立成 asmdef 时, 至少需要引用:

```json
"Excel2SO.Editor"
```

如果 importer 访问项目运行时类型, 还需要引用对应的项目程序集。当前 `TableImporter/Importers` 中的示例 importer 依赖 `Game.Core`、`Game.Items`、`Game.Gameplay`、`Game.Animation`。

## 常见问题

### 找不到 `Excel2SoListAssetImporter`

检查项目 Editor asmdef 是否引用了 `Excel2SO.Editor`。

### 编辑器窗口找不到 UXML 或 USS

检查 `Assets/UnityEasyWorkTools/TableImporter/Editor/UI` 下是否存在 `Excel2SoImporterWindow.uxml` 和 `Excel2SoImporterWindow.uss`。

### importer 找不到项目里的数据类型

检查 importer asmdef 是否引用了目标数据类型所在的运行时程序集。
